## 版本命名规范

遵循语义化版本控制 (Semantic Versioning):
- **主版本号** (Major): 不兼容的 API 修改或重大架构变更
- **次版本号** (Minor): 向下兼容的功能性新增
- **修订号** (Patch): 向下兼容的问题修正

## 更新类型标识

- 🎯 **重构** (Refactor): 大规模代码重构
- ✨ **新功能** (Feature): 新增功能
- 🐛 **修复** (Bugfix): 问题修复
- 🔧 **改进** (Enhancement): 功能改进
- 📚 **文档** (Documentation): 文档更新
- ⚡ **性能** (Performance): 性能优化
